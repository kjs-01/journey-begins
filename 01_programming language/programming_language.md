**프로그래밍 언어란:** 프로그램 작성을 위한 규칙의 집합

**프로그램:** 입력으로 부터 출력을 얻기 위한 명령어의 집합 <br>
**프로세스:** 실행 중인 프로그램 <br> 
**프로세서:** 프로세스를 작성하는 도구

### 프로그램의 역할 
- 생각대로 기계를 작동하기 위해
- 문제 해결 아이디어를 제시하기 위해

### 프로그래밍 언어에서 가장 중요한 세 가지
- 표현력
- 유지보수성
- 실행 가능성

### 프로그래밍 언어에서 또한 중요한 세 가지
- 직교성: 언어의 개별 특성을 자유롭게 조합할 수 있는 것
  - 반례: C 언어 구조체 및 배열의 인수 전달
- 추상화 지원 : 세부 사항을 무시하고 간략하게 정보를 전달할 수 있는 것 (정보 은닉)
- 동형성(homoiconicity): 코드와 데이터를 하나로 표현할 수 있는 것
  - 예시 언어: LISP

### Fortran
IBM 704를 제어하기 위한 프로그래밍 언어
- 최초로 널리 사용된 고급 언어
- 초기 버전의 정적 할당: 단순하지만 유연하지는 못함
- 추상화 기능 부족
- 구문도 고정된 형태

### LISP (LISt Processing language, MIT)
- 기호 계산: AI를 위해 필수적임
- 두 개의 자료형: 아톰(atom)과 리스트
- 함수형 프로그래밍: 연산은 적용(application) 뿐
- 자료 구조: 일반화 리스트  `(A B C D)` &nbsp; `(A (B C) D (E (F G)))`
- 괄호로 나타낸 리스트로 언어의 구문 표현

**Evaluation**
- 아직도 널리 사용됨
- 지금은 꽤 빠르게 수행됨(compilation)

### IAL(ALGOL 58, International Algorithm Language)
**Motivation**
- 수많은 언어가 기계 종속적
- 알고리즘을 기술하고 설명할 보편적인 프로그래밍 언어가 필요함
- Fortran을 사용하지 않는 이유 - 소유권이 IBM에 있음

### COBOL
- FLOW-MATIC : 12글자 지원 식별자(하이픈 사용 가능)
- 영어로 표현된 문장으로 계산을 나타냄
- 각 문장의 첫 단어는 동사
- 데이터와 코드 분리

### SIMULA(SIMULAition)
- 이산적 이벤트 처리를 위한 시뮬레이션 언어
- 이후 설계되는 **Smalltalk**, Objective-C, C++ 등에 영향을 줌
- 클래스 지원 (데이터 추상화 지원)

### Smalltalk
- 객체지향의 개념(데이터 추상화, 상속, 동적 타입 바인딩)을 완벽히 구현한 최초의 프로그래밍 언어
- GUI 환경을 개척함
- Smalltalk으로부터 윈도우 시스템이 발전함
