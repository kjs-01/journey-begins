## 계층별 데이터 이동 단위

|이동구간|주요용어|설명|
|---|---|---|
|**Disk** <-> **Memory**|**Block**(또는 Sector)|하드디스크나 SSD가 데이터를 읽고 쓰는 물리적 단위(512B ~ 4KB)|
|**OS** <-> **Memory**|**Page**|가상 메모리 관리 시스템에서 사용하는 논리적 단위(4KB)|
|**Memory** <-> **Cache**|**Cache Line**|메인 메모리에서 CPU 캐시로 데이터를 한번에 긁어오는 단위(64B)|
|**Cache** <-> **CPUy**|**Word**|CPU가 한 번의 연산으로 처리할 수 있는 데이터의 단위(32-bit/64-bit == 4B/8B)|

- **Block(블록):** 주로 물리적인 저장장치(Disk)의 관점에서 단위
- **Page(페이지):** 주로 운영체제가 메모리를 관리하는 관점에서의 단위

### PCB의 'Block'은 데이터 크기를 말하는 걸까?
운영체제를 공부하면 프로세스에 대해서 다루게 된다. 이때 **PCB**(Process Control Block)이 등장한다.<br>
PCB의 'Block'은 데이터 전송 단위를 의미하지 않는다.

- **컴퓨터 구조에서의 Block:** "한 번에 4KB씩 옮기자!"라는 물리적 크기의 개념이다.(위에서 정리한 개념) 반면에,
- **PCB에서의 Block:** "프로세스 관리에 필요한 정보들을 이 **데이터 구조(구조체)**에 모아두자!"라는 논리적 단위의 개념이다.

즉, **PCB**(Process Control Block)는 **"프로세스 제어를 위한 정보들이 뭉쳐있는 데이터 블록(구조체)"** 라는 뜻이지, 
그 크기가 디스크의 1 Block(4KB 등)과 같다는 뜻은 아니다. 실제로 PCB의 크기는 운영체제마다 다르며, 수십 바이트에서 수 킬로바이트까지 다양하다.
