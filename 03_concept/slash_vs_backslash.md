## `/` (Forward Slash) vs `\` (Backslash) 차이
### 운영체제에서의 차이

| 구분 | `/` (Forward Slash) | `\` (Backslash) |
| --- | --- | --- |
| **사용 환경** | Unix 계열 (Linux, macOS), 웹 URL | Windows (전통적 방식) |
| **경로 예시** | `/home/user/documents` | `C:\Users\Name\Documents` |

> &nbsp; <br>
> **구분하는 기호가 다른 이유**<br><br>
> 옛날 DOS 시절, `/`는 이미 옵션(예: `dir /w`)을 나타내는 용도로 사용되고 있었다. 그래서 폴더를 구분하기 위해 남은 기호인 `\`를 선택하게 된 것이 오늘날까지 이어졌다. (최신 윈도우 환경에서는 많은 경우 `/`도 인식한다.) <br>&nbsp;
---
### C++ 언어에서의 차이
`/` (Forward Slash) 
- **산술연산자:** 나눗셈을 할 때 사용한다. (`int a = 10 / 2;`)
- **주석:** 코드 설명을 적을 때 사용한다. (`// 한 줄 주석` 또는 `/* 범위 주석 */`)
- **경로:** 문자열 내에서 경로를 적을 때, 가급적 `/`를 쓰는 것이 이식성(Windows/Linux 공용)면에서 좋다.

`\` (Backslash)
- **이스케이프 시퀀스:** 뒤에 오는 문자와 결합해 특수한 의미를 만듭니다.
  - `\n`: 줄 바꿈
  - `\t`: 탭(Tab) 공간
  - `\"`: 큰따옴표 그 자체를 문자로 인식

- **주의할 점:** C++ 문자열 내에서 진짜 백슬래시(\\)를 출력하고 싶다면 두 번(\\\\) 써야 합니다. 하나만 쓰면 컴파일러가 "어, 뒤에 특수문자 오나?"라고 오해하기 때문이다.
  
```cpp
// C++ 예시
string path1 = "C:/Users/Guest";    // 슬래시는 하나만 써도 OK
string path2 = "C:\\Users\\Guest";  // 백슬래시는 두 번 써야 하나로 인식
