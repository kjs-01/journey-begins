## 05_CPU Shceduling(Algorithms)

### Shceduling Algorithms
성능 지표 -> Turnaround time = Completion - Arrival

대시 시간 : Ready queue에서 기다린 시간

### FCFS Scheduling
First Come, First Served
- CPU를 요구하는 프로세스에게 먼저 CPU를 할당해줌
- First-In First-Out(FIFO)
- Nonpreemptive
- No starvation(fairness 하기 때문에)

### SJF Scheduling
Shortest Job First
말 그대로 이기 때문에 부연 설명 X
같은 Run-time이 존재하면 FCFS 사용
- preemptive 를 추가한 버전이 STCF(Shortest Time-to-Completion First)

### Round Robin(RR) Scheduling
각각의 프로세스는 time quantum(slice)의 cpu time을 가진다.
이후에는 ready queue의 가장 마지막에 배치된다.

각 프로세스는 순서대로 일정한 시간의 CPU 시간을 할당받는다.
time quantum은 프로세스 런 타임의 공약수 단위면 좋다.

### Priority Scheduling
같은 우선순위는 FCFS algorithms로 실행

### Multilevel Queue Scheduling
레디 큐가 여러 큐로 분할되고, 하나의 프로세스는 분할된 큐 하나에 배정된다.
이때 배정되는 기반은 프로세스의 특징에 따라 나뉜다.
각 분할된 큐는 개별 알고리즘이 적용된다.

MLQ의 목적
- turnaround time의 최적화
- 상호작용하는 작업의 반응 시간을 최소화

### Multilevel Feedback Queue Scheduling
여러 종류의 Queue가 있는데 위 순서대로 Queue가 empty해야 아래 queue 프로세스 실행
? 근데 프로세스는 여러 큐 사이를 이동할 수 있다며

### Incorporating I/O
Process A가 I/O request가 발생하여 blocked되었을 때, 디스크에서 작업을 수행할 때
Process B에 CPU를 할당해주는 방식을 의미한다.

### Real-Time CPU Schdeling
- Soft Real-Time systems: 중요한 작업으로서, 빠르게 실행되어야 한다.
- Hard Real-Time systems: 마감까지 반드시 실행되어야 한다.

Event latency(이벤트 지연 시간) that affect perfomance
1. Interrupt latency: interrupt가 도착하고 실행되기 까지의 인터럽트 서비스 루틴
2. Dispatch latency: 스케쥴러가 현재 프로세스를 정지하고 다음 프로세스를 시작시키는 데 걸리는 시간
   1. 인터럽트가 발생하고 처리하는 과정 <-> 실제 다음 프로세스가 실행하기 까지의 시간 사이가 dispatch latency

### Rate-Monotonic Scheduling
Period = Deadline,
Shorter period = higher priority

50 period 에 20 processing time
100 period에 35 porcessing time

0 20 50 70 75 ... 100
 p1 p2 p1 p2

이런식으로 period 사이에 우선순위와 작업 시간을 적절히 조합하여 스케쥴링하는 알고리즘

우선순위와 작업 시간에 따라 Missed Deadline 발생할 수 있음.
