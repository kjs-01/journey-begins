## 07_Memory Management(Virtualization)

### Address Space
실행 중인 프로세스에 대한 모든 정보가 담겨있는 공간으로  code, data, heap, stack으로 이루어져 있다.

- Code: 명령어가 존재하는 곳
- Heap: 동적 할당이 이루어지는 곳
- Stack: 반환값(주소), 지역 변수, 전달인자가 저장되는 곳

Address Space에서 주소 변환 메커니즘을 거치면 실제 메모리 주소로 변환된다.

모든 실행중인 프로그램의 주소는 가상이다. (실제 메모리 주소가 매칭되지 않는다.)

### Address Translation
하드웨어가 가상 주소를 실제 주소로 변환한다.
- 필요한 정보는 실제로 물리 주소에 저장되어 있다.

운영체제는 하드웨어를 관리하기 위해 특정한 포인트에 관여해야 한다.
- 운영체제는 신중한 개입을 위해 반드시 메모리를 관리해야 한다.
- 어느 부분이 해제됐고 사용되고 있는지 추적해야 한다.

세 가지 가정이 있다.
- 사용자의 주소 공간(프로그램의 모든 정보)이 계속해서 실제 메모리에 저장되어 있다.
- 주소 공간의 크기가 실제 메모리 사이즈보다 작다.
- 각각의 주소 공간은 같은 크기다.

```c
void func()
    int x;
    ...
    x = x + 3; // we are interested in
```

```assembly
128 : movl 0x0(%ebx) , %eax ; load 0+ebx into eax
132 : addl $0x03, %eax      ;  add 3 to eax register
135 : movl %eax, 0x0(%ebx)  ; store eax back to mem
```
아래와 같은 순서로 주소 변환이 이루어진다. 여기서 15KB는 Stack의 공간을 의미한다.
- Fetch instruction at address 128
- Execute this instruction (load from address 15KB)
- Fetch instruction at address 132
- Execute this instruction (no memory reference)
- Fetch the instruction at address 135
- Execute this instruction (store to address 15 KB)

### Single Relocated Process
주소 공간(16KB)을 통째로 물리 주소(64KB)에 올리는 것.

### Static Relocation
- 운영체제가 각 프로그램을 메모리에 로드하기 전에 다시 작성한다.
- 정적인 데이터와 함수의 주소가 변경된다.

장점
- 하드웨어의 지원이 필요없다.
  
단점
- 보안이 강화되지 않는다.
  - 프로세스가 운영체제 영역의 메모리에 피해를 끼칠 수 있다.
  - 다른 메모리 주소 영역을 읽을 수 있다.
- 배치되고 나면 주소 공간을 옮길 수 없다.
  - 외부 파편화로 인해 새로운 프로세스를 위한 공간을 할당할 수 없게 된다.
  
> 외부 파편화(external fragmentation)란 할당할 수 있는 전체 메모리 공간은 충분하지만 연속적이지 않아서 할당할 수 없다는 뜻이다. 반대로 내부 단편화(Internal fragementaion)은 할당받은 메모리가 필요한 메모리보다 많아서 낭비가 된다는 뜻이다.
>
> 여기서 external과 internal이라는 용어를 구분짓는 기준은 "할당된 구역의 경계선"을 기준으로 낭비되는 위치가 안쪽이냐 바깥쪽이냐 이다.

또한 뒷 부분을 공부했을 때의 지식으로는 외부 파편화가 일어나도 이를 극복할 수 있는 방안이 있었는데 현재는 Address Space를 통째로 Physical Memory에 할당하는 가정의 영역이기 때문에 이런 파편화가 일어날 수 있다. 

### Dynamic Relocation
하드웨어 기반 재배치
- MMU: 메모리 참조가 일어나는 모든 명령에 대해 주소 변환을 수행한다.
- 보호(안정성)가 강화된다. 만약 주소 공간이 유효하지 않을 경우 MMU가 예외를 발생시킨다.
- 운영체제는 현재 프로세스의 유효한 주소 공간의 정보를 MMU로 전달한다.

운영체제는 프로세스가 실행될 때 실제 메모리 주소의 어디에 load 되어야 하는가를 결정한다.

- physical memory = virtual address + base register
- 0 <= virtual address < bound


### Base and Bound
- Bound: 범위의 크기
  - Address Space에서는 주소 공간의 크기를 의미한다.
  - Physical Memory에서는 주소 공간의 끝을 의미한다.
- Base: Physical Memory의 최소한의 물리 주소

프로세스와 운영체제를 지키기 위해 필요한 개념이다.

### MMU
virtual to physical address

메모리로 보내지는 모든 주소값(사용자 프로세스에 의해 생성되는)에 base register 값을 더한다. 그렇다면 base 레지스터에 값은 종종 변하지 않을까. not in use는 종종 변해야 하는데 base register의 값이 큰 채로 고정이 되버리면 그 값보다 작은 메모리 주소 공간을 사용하지 못할 거니까.

### Memory Virtualzing
1. 프로세스 동작을 시작할 때문
   1. 물리 주소에 주소 공간이 들어갈 자리를 찾는다.
   2. 이때 사용 가능한 base register(free list) 순서대로 찾는다.
2. 프로세스가 종료되었을 때
   1. 메모리 회수
   2. 사용 가능한 free list에 넣는다.
3. context switch가 발생했을 때
   1. base-and-bound pair를 저장한다.
   2. blocked 된 프로세스의 PCB에 Base, Bound 주소를 저장한다.
